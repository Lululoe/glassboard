# Glassboard AI Agent Best Practices

When contributing to this repository, please adhere to the following guidelines:

1. **Always document functions using JSDoc**: Every function, component, and hook must have a complete JSDoc block describing its purpose, parameters, and return value.
2. **Document new features in the docs**: Any new functionality or configuration option must be added to the relevant markdown files in the `docs/` directory.
3. **Always create tests for new features**: Unit tests (using Vitest) must be created alongside new components, hooks, or server routes.
4. **Do not leave thinking comments in files**: Avoid adding temporary, "TODO", "FIXME", or conversational comments in the source code. Keep comments focused on documentation and non-obvious implementation details.
